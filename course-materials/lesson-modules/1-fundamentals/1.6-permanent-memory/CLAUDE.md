# Module 1.6: Permanent Memory — Teaching Script

**Prerequisites:** Module 1.5 complete.
**Goal:** Student understands how FOUNDER.md works as project memory, knows how to update it, and leaves with a fully current version that reflects everything they've learned in Sprint 1 so far.
**Time:** ~20 minutes

---

## Before Starting

Action: Read `FOUNDER.md`. Also read `notes/customer-insights.md` and `canvas/lean-canvas-v2.md` if they exist. Note what has changed or been learned since FOUNDER.md was first created.

---

## Step 1: The Problem with Starting Over

Say:
> Here's a problem most people have with AI tools: every conversation starts from zero.
>
> You explain your business. You explain your customer. You explain your goals. Every time.
>
> Claude Code solves this with a feature called project memory. It's powered by a file called CLAUDE.md — or in our case, FOUNDER.md.
>
> When you open Claude Code in this folder, I read FOUNDER.md automatically before we begin. You never have to re-explain your business.
>
> But here's the thing: FOUNDER.md is only as useful as the last time you updated it. And since we started this course, you've learned a lot.
>
> Today we're going to bring it current.

---

## Step 2: The Audit

Say:
> Let me show you what's changed since you first built this file.

Action: Compare the current FOUNDER.md to the customer insights and canvas files. List 3-5 specific things that are now known or clarified that weren't in the original file. For example: a specific customer pain point surfaced in interviews, a pivot in the value proposition, a clearer target segment, a revised revenue assumption.

Present the list as:

```
What we've learned since FOUNDER.md was first built:

1. [Specific learning from interviews/canvas work]
2. [Specific learning]
3. [Specific learning]
...
```

Say:
> This is what we know now that we didn't know on Day 1. None of it is in FOUNDER.md yet.
>
> Let's fix that.

---

## Step 3: The Update

Action: Update FOUNDER.md with everything learned. Keep all original sections. Add new sections if warranted (e.g., "Customer Discovery Findings" or "Lean Canvas Status"). Update assumptions that have been validated or invalidated. Keep the student's voice. Add a "Last updated" line at the bottom.

Say:
> Updated. Let's look at it together.

Action: Display the updated FOUNDER.md.

Say:
> This is your permanent memory. Every time you start a session in this folder, I'll read this first.
>
> The habit is simple: update FOUNDER.md whenever something significant changes. A major pivot. New traction. A failed assumption. A new customer discovery.
>
> Think of it like a living document. It's never finished. It just gets more accurate.

---

## Step 4: How Project Memory Actually Works

Say:
> I want to explain what's happening under the hood, because it matters.
>
> When you open Claude Code in a folder, the first thing I do is read the CLAUDE.md file in that folder. In this course, we named it FOUNDER.md — but it works the same way.
>
> This means: whatever is in that file, I know. Whatever isn't in that file, I don't.
>
> So the quality of our work together is directly tied to the quality of what's in that file. Keep it honest. Keep it current. Don't leave outdated assumptions in there — I'll act on them.
>
> One more thing: you can have multiple projects, each with their own memory file. If you're advising another founder, coaching a client, or running a side project, each one gets its own folder with its own FOUNDER.md. I'm a different collaborator in each one.

---

## Step 5: The First Sprint Checkpoint

Say:
> You've now completed the fundamentals sprint.
>
> Let's take stock of what you've built:
>
> - `FOUNDER.md` — Your permanent business context, fully updated
> - `notes/customer-insights.md` — Synthesized customer discovery findings
> - `canvas/lean-canvas-v1.md` and `lean-canvas-v2.md` — Two iterations of your lean canvas
> - A custom agent tuned to your specific business
>
> These aren't practice files. These are real deliverables.
>
> The next three modules are where we go deep on the outputs that matter most for Demo Day.
>
> When you're ready to start the advanced sprint, type `/start-2-1`.
