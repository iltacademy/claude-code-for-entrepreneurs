# Module 1.2: Visualizing Your Business — Teaching Script

**Prerequisites:** Module 1.1 complete.
**Goal:** Student sets up a visual editor (VS Code or Obsidian) alongside the terminal so they can see their files rendered while Claude builds them.
**Time:** ~15 minutes

---

## Before Starting

Action: Read `FOUNDER.md`.

---

## Step 1: The Problem with the Terminal Alone

Say:
> Right now, everything we create lives in files on your computer. But the terminal only shows you text.
>
> The move is to open a second pane — a visual editor — so you can watch the files take shape in real time as I build them.
>
> Think of it like this: I'm in the terminal, doing the work. You're watching it happen in a window where everything renders cleanly.
>
> You have two good options: **VS Code** (free, widely used, great for files and folders) or **Obsidian** (free, great for networked notes and markdown).
>
> Which one do you already have installed, or which do you want to use?

Check: Student names their choice or says they need to install one.

---

## Step 2a: VS Code Setup (if they chose VS Code)

Action: Only run this branch if student chose VS Code.

Say:
> Perfect. Here's what to do:
>
> 1. Open VS Code
> 2. Go to File > Open Folder
> 3. Navigate to this course folder and open it
>
> You'll see the full file tree on the left. Every file we create together will appear there.
>
> Let's test it.

Action: Create a file called `notes/hello-visual.md` with the content: `# This is your visual workspace\n\nYou can see this file in VS Code right now. When Claude creates files, they'll appear here in real time.`

Say:
> Look at your VS Code window. You should see `hello-visual.md` in the notes folder.
>
> Click it. See how it renders?
>
> That's the setup. Terminal on one side, VS Code on the other. You'll use this layout for the rest of the course.

---

## Step 2b: Obsidian Setup (if they chose Obsidian)

Action: Only run this branch if student chose Obsidian.

Say:
> Perfect. Here's what to do:
>
> 1. Open Obsidian
> 2. Create a new vault OR open an existing vault
> 3. Point it at this course folder
>
> Every `.md` file we create will appear as a note in Obsidian, beautifully rendered.
>
> Let's test it.

Action: Create a file called `notes/hello-visual.md` with the content: `# This is your visual workspace\n\nYou can see this file in Obsidian right now. When Claude creates files, they'll appear here as notes.`

Say:
> Check Obsidian. You should see `hello-visual.md` as a note in your vault.
>
> That's the setup. Terminal on one side, Obsidian on the other. This is your workspace for the rest of the course.

---

## Step 3: The Power of This Setup

Say:
> Here's why this matters.
>
> When we build your lean canvas in Module 2.1, you'll watch it fill in section by section in real time. When we draft your pitch in Module 2.3, you'll see it take shape as a clean formatted document.
>
> The terminal is where the thinking happens. The visual editor is where you see the output. Together, they make this feel like having a very fast, very smart writing partner.
>
> One tip: keep both windows open at the same time. Split your screen if you can. The best workflow is terminal on the left, visual on the right.

Check: Student confirms they can see their files in the visual editor.

---

Say:

>
> You're going to bring me customer conversations — or we'll create practice ones — and I'm going to synthesize them into insights you can act on.
>
> This is the skill that directly feeds your customer discovery deliverable.
>
